MP1 Writeup

Overview and Usage

  1. For this project, I built a Chrome extension that replaces the user's cursor with a Rilakkuma. To run the code, you will need to follow these steps:
    1. Download the code for the extension (File name is CryptoBase).
    2. Open the Chrome Extensions page.
    3. Turn on "Developer mode" in the top right corner of the page.
    4. At the top, click the "Load unpacked" button.
    5. Locate where on your computer the extension was saved, and upload the file.
    6. Click the puzzle icon at the top of your browser window, next to the website URL.
    7. Pin "Rilakkuma Buddy" by clicking the pin button.
    8. Navigate to a new website that isn't Google.
    9. Click the Rilakkuma Buddy icon, and enjoy your new Rilakkuma cursor!

Development Process

I broke up MP1 into several steps to complete the project:

  1. For this project, I first had to download the extension toolkit that was provided to us on GitHub.
  2. I played around with the sample extensions provided to get a sense of how the pieces were all working together to create this final product that the user experiences.
  3. Then, I created my own folder to keep the contents of the extension.
  4. In this folder, I created several new files, including a manifest.json, popup.html, popup.css, and popup.js. I also added all of the images that were needed for this extension.
  5. I updated the manifest.json to represent the extension, and wrote the HTML, CSS, and JavaScript code in each file, debugging along the way.
  6. I ran into an error in the middle, which was that my code only worked when I opened it on Live Server and not as an actual Chrome Extension. Here is when it was opened in the Live Server:
  7. Lastly, I uploaded my extension into Google Chrome in the developer mode to test it out, and wrote out the MP1 writeup.

Issue Deep-Dive

An issue I ran into during MP1 was getting my Chrome extension to work. While the extension worked in my live server, it was not working as an actual Chrome extension previously.

To fix this issue, I had to set the cursor to be the image of the Rilakkuma in the popup.css file, and I commented out a lot of unnecessary code. I also added the popup.js file into the "content_scripts" section of the manifest.json file.

Finally after all this, it started working as expected.
